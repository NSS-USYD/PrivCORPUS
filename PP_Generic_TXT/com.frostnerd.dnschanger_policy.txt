This short section is meant as a summary of the below legal text in plain english. However, it is not a replacement for reading the full policy below.
DNSChanger does not collect any personal data in the closer sense (your name, e-mail, Android ID, ...). It however collects crash logs and sends them to the developer (me, Daniel Wolf). By default those crash logs contain only the cause of the crash and no device data at all. You can however opt-in to a more exhaustive crash reporting which also reports some basic device data, like your Android version, device model, language and an ID assigned to your device (which is randomly generated on every install, but only sent after opt-in). Either way, this crash reports are send via an encrypted connection and processed using a self-hosted service (Sentry). No third-party ever has access to any of this data.
Because of the nature of this crash reporting I cannot trace back crash logs to any specific user (even after opt-in). As such, if you want me to delete your data I may not be able to, unless you provide me with the randomly generated crash ID - which you can find in the about screen in the app. If you did not opt-in to the full crash reporting (or specifically disabled it) there is no data assigned to crash reports at all and thus I can never delete those.
DNSChanger contains the Google Play services which are already part of your device and collect a lot of data for Google. I hold no control over this service and this data would already be collected without DNSChanger using it.
DNSChanger is specifically Open source so that people knowledable in Android programming can check on all these promises.
